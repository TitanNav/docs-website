[New Relic CodeStream](/docs/codestream/how-use-codestream/performance-monitoring#code-level) 을 Node.js 에이전트와 통합하여 상황에 맞는 APM 데이터를 IDE에 직접 표시합니다. 이 통합은 편집기의 코드 수준 프로덕션 원격 측정을 코드 작성 및 검토로 시각화합니다.

## 시작하다 [#get-started]

먼저 지원되는 IDE에 New Relic CodeStream 확장을 [설치](/docs/codestream/start-here/install-codestream) 하고 로그인합니다.

<Callout variant="important">
  New Relic CodeStream 통합은 [Node.js 에이전트 버전 9.7.5 이상](/docs/release-notes/agent-release-notes/net-release-notes) 에서 사용할 수 있으며 기본적으로 비활성화되어 있습니다. 이 구성을 변경하려면 [설명서](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration#code-level-metrics) 를 확인하십시오.
</Callout>

## 에이전트 속성 [#attributes]

Node.js 에이전트는 애플리케이션의 [웹 프레임워크](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent#support) 에서 생성된 대부분의 범위에 다음 특성을 보고하고 연결합니다.

<Callout variant="important">
  경우에 따라 에이전트는 애플리케이션 내에서 호출되고 IDE 내에는 표시되지 않는 기본 프레임워크 코드를 직접 래핑합니다.
</Callout>
