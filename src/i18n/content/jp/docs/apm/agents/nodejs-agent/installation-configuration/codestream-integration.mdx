[New Relic CodeStream](/docs/codestream/how-use-codestream/performance-monitoring#code-level)を Node.js エージェントと統合することで、状況依存の APM データを IDE に直接表示します。この統合により、エディターでコード レベルの運用テレメトリが、記述およびレビュー コードとして視覚化されます。

## 始めましょう [#get-started]

まず、サポートされているIDEにNew Relic CodeStream拡張機能を[インストール](/docs/codestream/start-here/install-codestream)し、ログインします。

<Callout variant="important">
  New Relic CodeStream 統合は、 [Node.js エージェント バージョン 9.7.5 以降](/docs/release-notes/agent-release-notes/net-release-notes)で利用でき、デフォルトでは無効になっています。この構成を変更するには、 [ドキュメント](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration#code-level-metrics)を確認してください。
</Callout>

## エージェントの属性 [#attributes]

Node.js エージェントは、次の属性を報告し、アプリケーションの[Web フレームワーク](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent#support)によって生成されるほとんどのスパンにアタッチします。

<Callout variant="important">
  場合によっては、アプリケーション内で呼び出され、IDE 内に表示されないベース フレームワーク コードをエージェントが直接ラップします。
</Callout>
