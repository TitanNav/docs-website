標準の分散トレースと[無限トレース](/docs/distributed-tracing/infinite-tracing/introduction-infinite-tracing)のどちらを使用するかを決定する前[に、サンプリングタイプに関するドキュメントを確認する](/docs/distributed-tracing/concepts/how-new-relic-distributed-tracing-works)ことをお勧めします。また、現在New Relic APMエージェントを使用していて、分散トレースを有効にしたい場合は、[プランニングガイド](/docs/distributed-tracing/concepts/distributed-tracing-planning-guide)を参照してください。

始める準備はできていますか？まだお持ちでない場合は、[New Relicアカウントにサインアップしてください](https://newrelic.com/signup)。永久無料です。

ディストリビューショントレースを設定するには、一般的に3つのステップを行います。

<img
  title="Diagram showing the three steps to set up distributed tracing."
  alt="Diagram showing the three steps to set up distributed tracing"
  src={distributedtracingHighLevelDistTr}
/>

1. **Identify services:** このリクエストを完了するために使用されているエンドポイント、サービス、言語、システムを特定して書き留めます（この情報は次のステップで必要になります）。以下のような環境図があれば、それを使ってリクエストを処理するサービスのリストを作成することができます。

<img
  title="A sample environment diagram showing how you might identify services."
  alt="A sample environment diagram showing how you might identify services."
  src={distributedtracingDistributedTracingMap}
/>

2. **サービスをインストゥルメントする：** トレースデータを送信できるように、指定した各サービスをインストゥルメントします。APM エージェントのように、サービスを自動的にインスツルメンテーションするツールもあれば、サービスに何らかのコードを挿入する必要があるツールもあります。インスツルメンテーションの手順については、以下のアイコンをクリックしてください。

3. **トレースの表示：** サービスをインストルメントした後、アプリケーションでトラフィックを発生させ、 [New Relic UI](https://one.newrelic.com/launcher/distributed-tracing.launcher) にアクセスしてトレースデータを確認します。
